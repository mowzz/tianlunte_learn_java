From OS point of view

  1. 操作系统有多种内存管理算法,   这个和硬件实现紧密结合,   不同硬件实现适合不同内存管理算法

  2. CPU根据program counter的值从内存中加载指令,   也就是说程序是被提前加载到内存中的

Memory content overview

  1. The first a few hundred or so (so-called Low-memory), stores a table of pointers (also called interrupt-vector) to interrupt handler routines

  2. Program Instructions The CPU can only load instructions from memory, so any program to run must be stored there
